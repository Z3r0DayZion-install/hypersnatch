## 2️⃣ ASK BEFORE YOU BUILD

### Message Template (Short, Not Salesy)

**Subject:** Quick question about offline forensic analysis

**Body:**
Hi [Name],

I'm building a lightweight offline tool for reconstructing links and artifacts from saved HTML/HAR files for forensic documentation.

Quick question:
When you're working offline on seized data, how do you currently extract and document accessible endpoints?

Is that workflow painful or fine as-is?

No pitch - just trying to understand real workflows.

Thanks,
[Your Name]

### Alternative Approaches

**Reddit Post (r/computerforensics):**
"Question for independent consultants: How do you handle offline link extraction from HTML/HAR files? Current tools seem either too basic or enterprise-level expensive."

**LinkedIn Post:**
"Digital forensics consultants: What's your workflow for extracting endpoints from offline HTML files? Looking to understand real-world pain points."

**Direct Message Strategy:**
- Personalize based on their recent posts/profile
- Reference specific challenges they've mentioned
- Keep it under 100 words

## 3️⃣ WHAT YOU'RE LOOKING FOR

### Positive Indicators (5+ responses needed)

**Pain Acknowledgment:**
- "Yeah, that's annoying."
- "We use X but it's clunky."
- "We build scripts for that."
- "Enterprise tools are overkill."
- "Manual documentation is time-consuming."

**Specific Pain Points:**
- "I spend hours manually extracting links."
- "Current tools don't work offline."
- "Evidence documentation is tedious."
- "Free tools lack professional output."
- "Enterprise tools are too expensive for my practice."

### Negative Indicators (Plan Flawed)

**No Pain:**
- "We already use Cellebrite/EnCase."
- "We don't need that."
- "We script it easily."
- "That's not a problem."
- "Our workflow works fine."

**Alternative Solutions:**
- "I use [specific tool] and it works great."
- "We have custom scripts that handle this."
- "Our firm provides enterprise tools."

## 4️⃣ THE PAINFUL QUESTION

### Self-Assessment Questions

**Market Need:**
- If I didn't build this, would these people even notice?
- Are they solving this problem with existing tools?
- Is this a real pain point or a nice-to-have?

**Personal Bias Check:**
- Am I building this because I enjoy the technical challenge?
- Am I solving my own problem instead of market problem?
- Am I over-engineering for a non-existent need?

**Reality Check:**
- How many people actually work offline with HTML/HAR files?
- Is the market large enough to support even a small business?
- Are there existing free alternatives that are "good enough"?

## 5️⃣ MICRO-PROOF TARGET

### Minimum Validation Threshold

**Before 90-Day Build Sprint:**
- **5 real professionals acknowledge pain** (specific problem, not general interest)
- **2 say they would test it** (willing to try beta version)
- **1 says they would pay if it works** (commitment to purchase)

### Success Metrics

**Positive Validation:**
- 5+ "pain" responses
- Clear articulation of current workflow problems
- Specific examples of time/money wasted
- Willingness to try new solution

**Failure Indicators:**
- <3 "pain" responses
- "Not a problem" responses
- Existing solutions mentioned as sufficient
- No willingness to test or pay

### Validation Timeline

**Week 1:** Reach out to 20 professionals
**Week 2:** Follow up with non-responders
**Week 3:** Analyze responses and make decision

## 7️⃣ ALTERNATIVE PATH IF MARKET FAILS

### Potential Pivots

**Technical Journalists:**
- Need: Web evidence preservation for stories
- Pain: Unstable web content, need reliable documentation
- Price Point: $49-$99

**Researchers:**
- Need: Archiving unstable web content
- Pain: Content disappears, need structured capture
- Price Point: $99-$149

**OSINT Hobbyists:**
- Need: Structured data extraction from web captures
- Pain: Manual extraction is time-consuming
- Price Point: $29-$49

**Legal Researchers:**
- Need: Structured capture logs for legal proceedings
- Pain: Manual documentation is error-prone
- Price Point: $149-$199

### Pivot Strategy

**Do Not Pivot Until:**
- Market validation is complete
- Current hypothesis is proven false
- Alternative market shows clear pain points

**Pivot Process:**
1. Validate current market (forensic consultants)
2. If failed, validate alternative market
3. Repeat until finding real pain
4. Only then build solution
