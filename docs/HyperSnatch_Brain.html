📋 Executive Summary

Project Overview

HyperSnatch is a Link Resurrection Engine operating inside an Offline DOM Workbench. It brings buried links back into visibility by analyzing static HTML snapshots and HAR captures, operating under strict compliance boundaries.

Core Positioning

  • NOT a scraper, bypass tool, or circumvention system
  • IS an offline DOM workbench for link resurrection
  • COMPLIANT with Cash Policy Shield and legal requirements
  • TRANSPARENT with full audit trails and evidence logging

Batch Processor (core/batch-processor.js)

Features:
- Concurrency control (max 5 items)
- Progress tracking with callbacks
- Item normalization and deduplication
- Error handling and retry logic
- Result aggregation

Key Methods:
- addItems(items) - Add items to queue
- start() - Begin batch processing
- processItems() - Handle concurrent processing
- normalizeItem(item) - Clean and standardize input
- getStatus() - Get current processing status

Input Detector (core/input-detector.js)

Features:
- Multi-type input detection (URL, HTML, HAR, text)
- Block separation with various delimiters
- Content analysis and metadata extraction
- Validation and processing hints

Key Methods:
- detectAndParse(content) - Main detection entry point
- separateBlocks(content) - Split into logical blocks
- detectBlockType(block) - Classify content type
- validateInput(detected) - Validate detected input
- getProcessingHints(detected) - Suggest processing strategy
